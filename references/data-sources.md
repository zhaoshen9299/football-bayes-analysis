## 1. 搜索前检查

每场比赛重新确认当前日期、比赛状态和数据源可用性。任何“推荐源”都可能改变覆盖、付费墙、接口或口径；不要因为它出现在本文件中就假设当前仍可用。

搜索时优先打开原始页面，不把搜索摘要当证据。记录发布/更新时间和抓取时间。遵守站点条款、robots、许可和速率限制；遇到反爬时改用合法替代源，不绕过访问控制。

## 2. 来源优先级

1. **赛事与俱乐部官方**：赛程、场地、名单、停赛、发布会、正式伤情、比赛规则。
2. **原始数据供应商或正式数据端点**：比赛级事件、xG、赔率快照。
3. **可靠通讯社与长期跟队记者**：官方未明确的伤情、训练和预计首发；标记为 `REPORTED`。
4. **聚合站与社区**：只用于发现线索或交叉检查，不单独支撑高影响结论。

来源“看起来权威”不等于字段权威。俱乐部最适合确认名单，但不一定适合估计球员影响；赔率 API 适合价格，不适合解释价格变化原因。

## 3. 各字段建议来源

| 字段 | 首选 | 校验规则 |
|---|---|---|
| 赛程、开球、场地 | 联赛/足协/赛事官网 | 再与俱乐部赛程核对；保存时区 |
| 赛制、加时、两回合 | 官方竞赛规程 | 区分 90 分钟与晋级 |
| 停赛 | 官方纪律公告、赛事名单 | 不依赖阵容聚合站 |
| 伤情 | 俱乐部公告/主帅发布会 | 媒体消息标记为 `REPORTED` |
| 官方首发 | 赛事或俱乐部官方比赛中心 | 预计首发不得写成确认 |
| 比赛结果和进球 | 官方/稳定历史数据库 | 检查补赛、弃赛和中立场 |
| xG/事件 | 同一供应商的比赛级数据 | 不跨供应商串联；记录模型版本 |
| 角球 | 官方比赛报告或同一事件供应商 | 保存双方逐场计数；排除加时或明确拆分；核对更正 |
| 赔率 | 带机构和采集时间的原始 API/页面 | 每家三项成组，逐家去水 |
| 角球盘口 | 带机构、盘口线和采集时间的原始页面/API | 同一机构的Over/Under成组；结算口径一致 |
| 天气 | 官方气象机构或场地附近站点 | 使用预报时刻，不用日均值 |
| 裁判 | 官方指派与历史比赛数据 | 小样本强收缩，不用刻板印象 |

可用的研究或历史数据示例：

- [StatsBomb Open Data](https://github.com/statsbomb/open-data)：公开部分赛事的比赛、事件、阵容和部分 360 数据；覆盖有限，发布分析时按其许可要求署名。
- [Football-Data.co.uk](https://www.football-data.co.uk/downloadm.php)：提供多联赛历史结果和赔率 CSV，适合回测；阅读同页 Notes 并检查字段年份。
- [The Odds API 文档](https://the-odds-api.com/liveapi/guides/v4/)：可获取按机构组织的赔率；历史快照能力和套餐会变化，调用前核对当前文档。
- Understat 等站点可作为覆盖范围有限的 xG 候选，但使用前检查当前可访问性、许可、赛事覆盖和数据口径。

Transfermarkt、聚合伤停站和赔率比较页只作为二级线索；关键缺阵至少用官方或可靠跟队来源确认。

## 4. 时效性

建议把快照分为：

- **早盘快照**：建模和初步赔率基准；首发多为预计。
- **赛前刷新**：重新抓取伤停、发布会、天气和赔率。
- **首发快照**：官方名单发布后重算；若用户需要最终版，明确预计刷新窗口。

不要假定首发总在固定分钟发布。每个快照都记录 `as_of`，比较时只比较同类字段。

赔率必须记录机构、市场、十进制价格和采集时间。无法确认是开盘、即时或收盘时，不贴标签。历史回测只能使用当时真实可用的快照。

## 5. 一致性与降级

- xG 模型会因射门位置、身体部位、助攻方式、门将和防守者位置等特征差异而产生不同数值。一个时间序列内保持供应商一致。
- Field Tilt、PPDA、禁区触球和“重大机会”各供应商定义不同；引用定义，不能只引用名称。
- 角球统计可能因事件更正、加时纳入方式和盘口结算规则不同而冲突；建模序列保持同一供应商和同一90分钟口径。
- 抓不到高级数据时，降级到同口径进球数据并加大收缩；不要用手工印象补数。
- 只有单家赔率时仍可去水，但把市场证据降为 B/C，并删除“市场共识”措辞。
- 关键数据冲突时并列展示，说明采用哪一个以及理由；不要偷偷择优。
- 来源不可访问时标记缺口并继续可完成部分，无需为了所有维度完整而编造替代值。

## 6. 已知可用性警告

以下是带日期的状态，不应永久视为真：

- **FBref（2026-01-20 公告）**：Sports Reference 表示其高级足球数据供应被终止并已移除相关数据；FBref 仍保留大量基础历史数据。因此不要把 FBref 固定写成当前 xG/高级指标默认源。每次使用前核验当前页面。[官方公告索引](https://www.sports-reference.com/blog/category/fbref/)
- **Football-Data.co.uk（页面当前警告自 2025-07-23 起）**：其页面称 Pinnacle 公共 API 赔率相对其他机构可能系统性过时，已不再用于市场平均/最大值计算。使用该字段回测前阅读最新 Notes。[下载与说明页](https://www.football-data.co.uk/downloadm.php)
- **StatsBomb Open Data**：只开放“部分赛事”，不能假设覆盖待分析联赛或最新赛季。[仓库说明](https://github.com/statsbomb/open-data)

在最终报告中只写与本场实际用到的数据源有关的警告。
