# 角球分析方法

## 预测口径

默认预测90分钟及补时内判罚的角球，不含加时和点球大战。先确认数据商与盘口结算规则；口径不一致时不得比较。

角球与进球是不同生成过程。阵地压迫、边路推进、封堵射门、落后后的比赛状态都可能增加角球，但不存在可跨赛事使用的固定换算率。

## 数据与角色

使用同一供应商逐场记录四个角色序列：

- 主队可比主场角球获得数；
- 主队可比主场角球被获得数；
- 客队可比客场角球获得数；
- 客队可比客场角球被获得数。

中立场国家队比赛优先使用同赛事、相近强度对手的中立场样本；样本不足时提高赛事基线的先验权重。保留日期、对手、场地、比赛状态和是否加时。不要只收集最近若干场而忽略可比性。

中立场的名义“主队”通常是赛程标签，不等于主场优势。赛事基线优先使用对称的每队角球均值，或在验证对阵排序偏差后显式加入实力/种子协变量；不要直接把名义主队均值和客队均值当成物理主客场基线。

## 收缩模型

对每个角色使用带时间权重的 Gamma–Poisson：

```text
lambda ~ Gamma(baseline * prior_weight, prior_weight)
w_i = reliability_i * 0.5 ** (days_ago_i / half_life_days)
alpha_post = alpha_0 + sum(w_i * corners_i)
beta_post = beta_0 + sum(w_i)
```

组合双方角球率：

```text
home_rate = home_for * away_against / baseline_home
away_rate = away_for * home_against / baseline_away
```

从四个后验抽样，再从对应 Poisson 抽取比赛角球。参数混合自然产生比固定 Poisson 更宽的预测分布。若有充足历史数据，应比较负二项、双变量计数模型和带比赛状态协变量的分层模型，并按时间滚动验证。

## 输出

至少报告：

- 主队角球均值及90%预测区间；
- 客队角球均值及90%预测区间；
- 总角球均值及90%预测区间；
- 总角球最可能的整数结果；
- 8.5、9.5、10.5、11.5等与当前盘口相关的大小概率；
- 主队角球更多、相同、客队更多概率；
- 同时点角球盘口的逐家去水共识（如可得）。

预测区间描述单场计数波动，不是参数可信区间。总角球期望高于盘口线不自动意味着“大角有价值”。

## 战术情景

只有经历史估计的映射才能调整角球率。否则只列具名情景：

- 强队持续压制、边路传中被封堵，角球上行；
- 强队早早领先并降低推进强度，总角球可能下行；
- 弱队落后后增加直接进攻，其自身角球可能上升；
- 边锋、边后卫或高点缺阵会改变进攻路径，但不能按名气手工加减角球。

## 校准

冻结主客角球的完整预测分布。赛后报告主客及总角球MAE、RMSE、Poisson/离散对数评分、各大小线Brier分数和预测区间覆盖率，并与同一截止时点的角球市场比较。至少积累30场后才讨论初步校准，分赛事结论需要更大样本。

赛后先从官方报告确认90分钟及补时角球，明确排除加时角球。若两个供应商的赛事总数或单场计数不一致，先建立冲突表并排除未解决比赛，不得为了凑样本择一使用。运行 `scripts/evaluate_corners.py` 计算均值误差、MAE、RMSE、90%区间覆盖率和各半球线Brier。

赛事内更新角球基线必须使用滚动截止：第 `n` 场只能使用前 `n-1` 场已经结束且口径确认的数据。阶段变化会改变比赛强度和状态分布；小组赛均值不能无收缩地替代淘汰赛基线。

回测必须同时保存联赛均值朴素基线，并对模型与基线的逐场绝对误差做配对区间。还应按球队汇总“预测总角球减实际总角球”残差，用于发现风格极端、升班马和结构突变；这些残差是下一轮建模线索，不是手工修正百分点的依据。
