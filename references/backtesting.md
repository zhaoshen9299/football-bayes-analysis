# 赛后复盘与回测协议

## 1. 先锁定评估对象

每条冻结记录只评估一个明确目标：

- `result_90min`：90分钟及补时的主胜、平、客胜；
- `to_qualify`：最终晋级方；必须使用独立的二项概率记录；
- 角球：90分钟及补时角球，不含加时。

淘汰赛若进入加时或点球，`result_90min` 的实际结果必为平局。不得用120分钟比分或晋级方替换。比赛报告必须分别保存 `home_goals_90`、`away_goals_90`、`result_type`、最终比分、晋级方与来源。

## 2. 建立完整性台账

先列出研究窗口内所有比赛，再逐场标记：

- `FROZEN_FORECAST`：存在开赛前冻结快照；
- `NO_FORECAST`：没有赛前概率，只能做事实复盘；
- `INVALID_FORECAST`：冻结时间、目标或来源无法验证；
- `PENDING_RESULT`：尚未完赛。

先报告覆盖率，再报告评分。不得只选命中的比赛；不得用赛后赔率、赛后阵容或比赛过程重建“赛前预测”。

## 3. 结算顺序

1. 使用官方比赛报告确认90分钟比分、加时/点球状态和角球口径。
2. 校验 `frozen_at < kickoff`。
3. 校验预测目标与实际字段一致。
4. 对相同截止时间的市场基准做成对比较。
5. 按阶段、数据层级、快照类型和预测提前量分层，但小样本只描述不归因。
6. 把赛果命中与概率质量分开：单场“猜中”不能证明概率正确。

## 4. 评分与样本量

胜平负使用 Log Loss、Multiclass Brier 和 RPS；命中率仅作辅助。角球使用主客及总数 MAE、RMSE、均值误差、90%预测区间覆盖率和各半球线 Brier。

- 少于30个独立比赛：不解释校准分箱或分层胜率；
- 30–99场：只报告初步方向；
- 100场以上：仍需给出赛事、阶段和时间稳定性限制；
- 同一比赛多个快照属于相关观测，不得增加独立比赛样本量。

只有在下一段未见比赛上改进，才视为模型升级证据。对刚复盘样本调参后的同样本指标不作为证明。

整季验证采用三段式时间切分：更早赛季训练、下一赛季验证并锁定参数、最后赛季只做一次测试。不得看完测试指标后再选择半衰期、先验权重、`rho`、特征或市场混合权重。模型与基准差异使用逐场配对重采样区间；区间跨过0时只报告点估计方向，不宣称稳定优势。

同一开球时间的比赛互相不可见。滚动程序必须先为该时间点所有比赛生成预测，再将这些比赛的结果批量加入历史；逐行预测后立即更新会造成赛程末轮等同时开球场次的数据泄漏。

## 5. 淘汰赛与中立场

- 分开建模90分钟结果和晋级概率；加时阶段需要体能、换人深度和剩余时间的独立模型。
- 中立场的“主队/客队”通常只是赛程排序。不要把名义主队角球或进球差当作主场效应；检查种子、排名与对阵生成规则造成的排序偏差。
- 赛事进行中更新基线时，只允许使用该场开球前已经结束的比赛，记录当时样本数；不得用整届赛事最终均值回填早期比赛。

## 6. 数据冲突

同一比赛的比分、90分钟/120分钟口径、角球或 xG 在供应商间冲突时：

1. 优先官方比赛报告；
2. 并列记录冲突值、供应商和获取时间；
3. 未解决前排除该字段的定量评分；
4. 不以“多数来源一致”替代口径核验。

## 7. 冻结记录示例

```json
{"match_id":"wc2026-qf-norway-england","target":"result_90min","stage":"Quarter-finals","data_level":"L3","snapshot_kind":"pre_match","forecast_role":"market","kickoff":"2026-07-11T21:00:00Z","frozen_at":"2026-07-11T03:09:13Z","probabilities":{"home":0.237546,"draw":0.255532,"away":0.506922},"actual":{"home_goals_90":1,"away_goals_90":1,"result_type":"aet","source":"official-match-report"}}
```

`evaluate_forecasts.py` 会从90分钟比分推导结果，并拒绝“加时比赛在90分钟非平局”等冲突记录。批量研究应通过 `--manifest` 提供完整比赛清单；晋级概率使用 `evaluate_qualification.py` 独立评分。
