# 报告模板

默认使用标准版。没有数据的字段删除或标记 `UNAVAILABLE`，不要用空洞文字填满。

```markdown
# {赛事}｜{主队} vs {客队}

- 开球：{本地时间 + 时区}（{UTC}）
- 口径：90 分钟常规时间｜{主客/中立/两回合说明}
- 分析截止：{as_of + 时区}
- 数据层级：{L1/L2/L3/L4}｜证据质量：{A/B/C}
- 一句话结论：{概率最大的结果 + 与市场是否一致 + 最大限制}

## 概率总览

| 结果 | 模型概率 | 90% 区间 | 去水市场概率 | 差异 |
|---|---:|---:|---:|---:|
| 主胜 | {x.x%} | {x.x–x.x%} | {x.x%} | {+/-x.x pp} |
| 平局 | ... | ... | ... | ... |
| 客胜 | ... | ... | ... | ... |

- 模型：{名称/模式、版本、rho、主要超参数}
- 市场：{机构数、赔率采集窗口、比例去水、中位 overround}
- 后验进球率：主队 {均值 [区间]}；客队 {均值 [区间]}
- 最高概率比分：{比分 概率}、{比分 概率}、{比分 概率}

> 若为 L3：删除模型列，只保留市场概率，并写“无独立模型概率”。

## 角球预测（用户要求时）

| 指标 | 后验均值 | 90%单场预测区间 |
|---|---:|---:|
| 主队角球 | {x.x} | {a–b} |
| 客队角球 | {x.x} | {a–b} |
| 总角球 | {x.x} | {a–b} |

| 总角球线 | 模型大球概率 | 去水市场大球概率 | 差异 |
|---|---:|---:|---:|
| 8.5 | {...} | {...} | {...} |
| 9.5 | {...} | {...} | {...} |
| 10.5 | {...} | {...} | {...} |

- 角球数领先：主队更多 {...}｜相同 {...}｜客队更多 {...}
- 最可能总角球：{整数及概率}
- 模型：{模式、赛事基线、先验权重、半衰期、抽样数}
- 口径：90分钟及补时，{不含/包含}加时；数据供应商 {名称}
- 限制：{样本可比性、比赛状态、阵容与盘口口径}

> 无同口径逐场数据时删除模型列，只报告角球盘口去水概率或 `UNAVAILABLE`。

## 从数据到后验

| 证据 | 状态/等级 | 进入模型的方式 | 影响 |
|---|---|---|---|
| {同口径历史表现} | {CONFIRMED/A} | {似然 + 时间衰减} | {影响进球率，不写主观百分点} |
| {伤停/首发} | {...} | {基础/情景/未纳入} | {...} |
| {赛程或场地} | {...} | {...} | {...} |

明确列出没有纳入模型的高噪声信息及原因。

## 比赛机制与情景

### 情景 A：{名称}

- 触发条件：{可观察条件}
- 机制链：{战术行为 → 机会数量/质量 → 进球率方向}
- 支持证据：{本场相关证据}
- 反证：{为什么可能不发生}
- 敏感性：{若有量化，给情景结果；否则只给方向}

### 情景 B：{名称}

同上。两个情景应互斥或明显不同，不要只是换一种叙述。

## 市场比较

- 共识公平概率：{...}
- 模型差异：{...}
- 结论等级：{一致 / 方向性差异但不足 / 经回测支持的显著差异}
- 价格失效条件：{赔率移动到何处或截止多久需刷新；没有验证时不写下注阈值}

## 风险与刷新条件

1. {最大的参数或数据不确定性}
2. {阵容/伤情状态}
3. {模型假设或供应商口径}

以下变化会使本报告失效：{官方首发、关键伤情、场地/天气、显著赔率变化等}。
建议刷新：{时间或事件触发条件}。

## 关键来源

- {来源标题｜发布时间/数据时间｜链接}
- {...}

概率是基于当前信息的估计，不是赛果或收益保证。
```

## 简版

用户只要结论时仍保留：比赛身份与截止时间、数据层级、胜平负概率、市场概率、最大不确定性和 3–5 个关键来源。不要省略概率口径。

## 刷新版

在标准版前增加：

```markdown
## 相比 {旧快照时间} 的变化

| 字段 | 旧值 | 新值 | 新来源 | 模型影响 |
|---|---|---|---|---|
| ... | ... | ... | ... | ... |

- 数据变化造成的概率变化：{...}
- 方法/参数变化造成的概率变化：{...}
```

不要把方法变化误写成外部信息变化。

## 多场检查点

```markdown
|---|---|---|---|---|
| ... | 已完成/待刷新/待处理 | ... | ... | ... |
```

## 赛后复盘版

先读取 `backtesting.md`，再使用：

```markdown
## 评估边界

- 研究窗口：{开始—结束}
- 已完赛：{N}｜有效冻结预测：{n}｜覆盖率：{n/N}
- 预测目标：{90分钟胜平负/晋级/90分钟角球}
- `NO_FORECAST`：{N-n}场，只做事实复盘，不纳入概率评分

## 逐场对齐

| 比赛 | 90分钟比分 | 最终状态 | 冻结概率 | 正确结算 | 单场Log Loss | 备注 |
|---|---:|---|---|---|---:|---|
| ... | ... | 常规/AET/点球 | ... | 主/平/客 | ... | ... |

## 汇总评分

报告 Log Loss、Multiclass Brier、RPS、市场成对基准、角球误差与样本量限制。L3预测不得包装成独立模型。

## 经验与修改

只把可重复、可前置执行的经验写入方法；区分数据修复、流程修复和模型参数变化。参数变化必须留给下一段未见比赛验证。
```
